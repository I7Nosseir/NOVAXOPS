Quick Reference — Essentials

  • Global search (top-right header) finds any task or client instantly.
  • Click any Pipeline card to open the full Task Detail panel.
  • Badge counts in the sidebar update automatically every 60 seconds.
  • AI agent responses are cached — same task + same context = instant result.
  • Esc key closes any open panel or modal.
  • Dark / light mode toggle is in the top header.
  • All documents auto-save as you type.
  • The floating "My Tasks" button is available from every page.

01 — Getting Started

First login, navigating the app, and understanding your role.

p.3
Logging In
  1. 1Go to the NOVAX Ops URL your admin shared and enter your email and password. First-time users receive an invite email from the system.
  2. 2On your first login you will be prompted to set your display name, phone number, and choose a secure password. Complete this before doing anything else.
  3. 3Once setup is complete you land on the Dashboard — your home base for the day.
Navigation Layout

Left Sidebar

  • NOVAX logo at the top — click to return to Dashboard from anywhere.
  • All page links with live badge counts (approvals, moderation).

  • Collapses automatically on small screens.

Top Header

  • Search bar — find any task, client, or project by name.
  • + New Task — create a task from any page.
  • Notifications bell — all recent activity across the platform.
  • Theme toggle — switch between dark and light mode.

The 10-stage Kanban that drives every piece of content from brief to reporting.

p.5

Core Pipeline Actions
  • Drag a card to a different column to move it through the pipeline.
  • Click a card to open the full Task Detail panel on the right.
  • Filter chips at the top filter by client, assignee, priority, or stage.
  • Advanced Filter panel lets you combine multiple conditions.
  • Task cards show: priority badge, client colour strip, due date, assignee avatar.
  • Overdue tasks show a red due date — action these immediately.
Filters persist. When you set a filter (e.g. "Client X only"), it stays active until you clear it. Remember to clear filters when switching to a different task or client.
Task Detail Panel — What's Inside

Opens when you click any task card. Contains:

  • Full title, description, priority, due date, and assignee.
  • Subtask checklist and task comments thread.
  • 5 AI Agent buttons (see below).
  • Copy versioning — AI outputs are saved and labelled per run.
The 5 AI Agents
Analyze Task Breaks down the brief, flags missing info, suggests an approach.
Write Copy Generates 3 caption variants: Aspirational, Benefit-led, Conversational.
Research Market context, competitor gaps, trending angles relevant to the task.
Find Assets Extracts keywords to query the asset library for relevant files.
Generate Caption Image-aware caption using the uploaded media as visual context.

04 — My Tasks  ·  05 — Clients

Your personal work queue and how to manage all client brands in one place.

p.6
My Tasks — What You See
  • Only tasks assigned directly to your account across all clients.
  • Sorted by due date — overdue tasks appear first in red.
  • Search bar filters by task title keyword.
  • Priority badges visible on each row: Critical / High / Medium / Low.
  • Click any row to open the Task Detail panel.
The floating My Tasks button in the top header is available on every page. Use it to check your queue without leaving your current page.
Creating a New Task
  1. 1Click "+ New Task" in the top header (available from any page).
  2. 2Fill in: title, client, pipeline stage, priority, assignee, and due date.
  3. 3Add a detailed description — this is what the AI agents read to generate relevant output.
  4. 4Click Save. The task appears immediately in the Pipeline.
Tip
A vague description like "write caption" produces generic AI output. A description like "Write 3 Ramadan caption variants for [Client X], targeting women 25–40, in a warm and conversational tone" produces work-ready output on the first try.
05 — Clients
Client Cards
  • Each client appears as a card with their brand colour.
  • Shows open task count and current stage distribution.
  • Red "CRISIS" badge appears if crisis mode is active.
  • Click any card to open the Client Detail modal.
Client Detail Modal — 3 Tabs
Overview Brand identity, social platforms, tone of voice, audience, design brief.
Intelligence Context bank (wins, objections, signals), AI feedback taste profile, competitor tracking.
Tasks All open tasks for this client, grouped by stage. Click any task to open it.

Crisis Mode

  • Activated per client by Admin or CEO only.
  • Adds a visible red "CRISIS" banner on the client card and all their tasks.
  • Signals the team to pause normal scheduling and await instructions.
  • Deactivate from the same toggle in the client detail modal once resolved.
Adding a New Client — 9-Step Wizard

Click "New Client" to launch the onboarding wizard:

  1. Identity → Social Platforms → Competitors
  2. Brand Guidelines → Tone of Voice → Audience
  3. Strategy Goals → Resources → Confirmation

The wizard runs an automatic SWOT analysis via AI after completion. A full client brief is ready before the first task is created.

Context Bank: Fill the Intelligence tab as you learn things — wins, objections, signals. This data is injected into every AI call for that client, making outputs progressively smarter.

06 — Projects  ·  07 — Publishing

Campaign grouping and scheduling content across all clients and platforms.

p.7
Projects — Campaigns & Initiatives

Projects group related tasks under one campaign. Use them for any initiative with 5+ tasks.

  • Project card shows progress bar (% of tasks completed).
  • Task count breakdown by stage visible at a glance.
  • Filter the Pipeline by project to see only that campaign's tasks.
  • Archive completed projects — the data is valuable for future reporting.
Example Setup
"Ramadan Campaign 2026 — Client X": 1 strategy task, 6 copy tasks, 6 design tasks, 1 approval task, 6 scheduled tasks. Progress bar shows 8/20 complete (40%). Everyone on the team knows exactly where the campaign stands without a separate update.

07 — Publishing
Two Views

Grid View

All scheduled posts as cards. Filter by client and date range. See status per post: Scheduled / Published / Failed. Click any card to view its caption and media.

Calendar View

Month or week calendar with posts plotted by scheduled publish time. Shows content frequency per client. Use this to spot gaps and over-crowded days before the week begins.

Generate Calendar: Click this button to have AI suggest a full posting schedule for a client — topic ideas per day based on their strategy and content pillars.
Composing a Post
  1. 1Click "Compose" and select the client and target platform(s).
  2. 2Choose language: English, Arabic, or Both. "Both" generates two separate caption versions.
  3. 3Write the caption manually, or click "Generate" to have AI draft one using the client's brand voice and context bank.
  4. 4Upload media (image or video), or link from the Asset Library.
  5. 5Set the publish date and time, then click "Schedule" to send to the scheduling platform.
Failed posts show a red status badge. The root cause is always in the scheduling platform — check there for error details (usually a media format issue or expired token).

08 — Approval

Send content for client review via a secure shareable link — no login required for clients.

p.8
Creating an Approval Request
  1. 1Go to Approval and click "New Request".
  2. 2Select the client and choose which posts to include in this batch.
  3. 3The system generates a unique approval link and token for this batch.
  4. 4Copy the link and send it to the client via email or WhatsApp.
  5. 5The client opens the link — no account, no password, no app to install.
  6. 6You receive a notification as soon as the client submits their review.
Internal Approval View
  • Lists all pending requests across all clients in one view.
  • Shows which posts are waiting, approved, or have requested changes.
  • Filter by client to focus on one brand at a time.
  • Add internal notes on any request — only the team can see these.
  • Multiple approval requests can be open simultaneously per client.
What the Client Sees (Client Portal)

The client opens a branded approval page showing each post with its media, caption, and scheduled publish date. For each post they can:

Approve Marks the post as approved and ready to publish.
Request Changes Flags the post and lets the client add a note explaining what to change.
Add Note Client can leave a general comment on any post without changing its approval status.

After reviewing all posts, the client clicks Submit. You receive a notification immediately. Posts with "Request Changes" move back to the Review stage in the Pipeline.

Send approval links 48 hours before the scheduled publish time. This gives the client time to review and allows you to complete revisions before the post goes live.
Real-world workflow
Account Manager batches this week's 6 posts for Client X into one approval request. Sends the link on Sunday. Client reviews on Monday morning, approves 5 posts, requests changes on 1. Account Manager sees the notification, opens the request, reads the client's note, updates the task in the Pipeline, hands it back to the copywriter. Revised post is approved on Tuesday and goes live Wednesday as planned.

09 — Moderation  ·  10 — Assets

Comment queue with AI-drafted replies, and the centralised file library.

p.9
The Comment Queue
  • Aggregates comments and DMs received via the messaging platform.
  • Sorted by time received — oldest unanswered items first.
  • Filter by client or by status: Pending / Replied / Ignored / Escalated.
  • Each item shows: commenter name, platform icon, full message text.
Actions Available Per Item
Generate Reply AI drafts a reply in the client's tone of voice. You edit it before sending.
Send Reply Sends your reply to the messaging platform and marks the item as replied.
Escalate Flags the item for a senior team member to handle.
Ignore Archives the item without replying (e.g. spam, irrelevant).
Always read the AI draft before sending. Check for factual accuracy and brand tone. The AI reads the client's context bank, so the more you fill that in, the better the drafts get over time.
Platform limitation: Instagram public comment replies are not supported via the messaging platform API — only DMs. Facebook public comments do work. Google Reviews are not supported.
10 — Assets

Centralised file library for all client creative assets — images, videos, PDFs, and documents.

Direct Upload

  • Drag and drop files onto the upload zone, or click to browse.
  • Supports: images (JPG, PNG, WEBP, GIF), video (MP4, MOV), PDF, documents.
  • Up to 500 MB per file.
  • Tag each asset to a client for easy search later.

Google Drive Import

  1. 1Click "Import from Drive" — on first use, authorise Google Drive access.
  2. 2Browse your Drive folders and select files to import.
  3. 3Files are copied into the asset library — no ongoing sync is needed.
Naming Convention Tip
Name files descriptively before uploading: ClientX_Ramadan_Reel_v3.mp4 rather than final_FINAL_2.mp4. This makes search and linking to tasks significantly faster. The asset library has no folder structure — file names are your only navigation tool.

11 — Creative Eval  ·  12 — Workload

AI scoring for creative assets before they reach the client, and team capacity management.

p.10
Creative Eval — AI Scoring

Upload a creative asset — image or video — and receive a scored evaluation across 4 dimensions within 30 seconds.

  1. 1Upload an image or video onto the upload zone.
  2. 2Select the client. Optionally add a note about the creative goal (e.g. "awareness reel for Ramadan").
  3. 3Click "Evaluate". The AI analyses the asset in context and returns scores.
What Gets Scored (0–100 per dimension)
Brand FitDoes it match the client's visual identity, colour palette, and tone?
Hook StrengthWill it stop the scroll in the first 3 seconds?
Visual QualityTechnical quality, composition, lighting, and clarity.
CTA & EngagementDoes it drive a clear action or invite interaction?
Use Creative Eval before sending work to the client. If any score is below 65, revise the asset internally first — a rejected approval wastes 2–3 days of the production timeline.
12 — Workload

A visual overview of how many tasks each team member is carrying right now versus their capacity.

What You See

  • One row per team member with avatar, name, and role.
  • Progress bar: current task load vs capacity threshold.
  • Green = healthy  ·  Amber = near capacity  ·  Red = overloaded.
  • Expand any row to see that person's assigned tasks with priority badges.
  • Click any task from Workload to open its Task Detail panel.
How to Use It Effectively
  • Check this page before assigning a new task to avoid overloading a teammate.
  • If someone is overloaded, reassign one of their lower-priority tasks to a teammate with capacity.
  • Use it in weekly team meetings to balance the week's workload before it begins.
  • Creative Directors and Admins see all team members. Other roles see only their own row.
Monday standup tip
Open Workload on the shared screen during standup. If any bar is red, the Creative Director can immediately reassign tasks — before the bottleneck delays a client deliverable.

  1. 1Go to Settings → Team and click "Invite User".

Page Permissions

Control which pages each user can access from the sidebar. This lets you customise each role's workspace.

NULL (default) User sees all pages they are allowed by their role.
Empty array User sees only the required pages (Dashboard, Pipeline, My Tasks).
Custom array Exactly the pages listed are visible. Everything else is hidden.

Activity tab: If AI costs are unusually high, check which user is generating them. The rate limit is 10 AI requests per user per minute — accidental repeated runs can accumulate cost quickly.

17 — AI Assistant

A context-aware chat assistant that reads your clients, tasks, and brand rules. Accessible by all team members.

How It Works
  • Open from the sidebar under "AI Assistant".
  • Select a client to scope the conversation to that brand. Without a client selected, the AI gives generic answers.
  • Optionally link to a task for task-specific help.
  • The AI reads: client brand identity, tone of voice, context bank, recent AI feedback, and current user role.
  • Conversation history is remembered within the current session.
  • Start a new chat at any time to clear context and begin fresh.
Always select the client first. The difference in output quality between a scoped and unscoped session is significant — scoped sessions produce brand-accurate, immediately usable output.
What to Ask the AI Assistant
For copywriters
"Draft 3 hook options for a Ramadan reel for [Client]."  ·  "Rewrite this caption in a more conversational tone: [paste text]."  ·  "Give me 5 caption angles for a product launch targeting women 25–35 in the Gulf."
For account managers
"What are the key brand rules I should follow for [Client]?"  ·  "Summarise the main competitor gaps we have in this space."  ·  "What are this client's stated goals and what should we be prioritising?"
For strategists
"Suggest 5 content pillar topics for a Q3 Ramadan campaign for [Client]."  ·  "What's a strong cultural tension we could use for a campaign in this market?"  ·  "Give me 3 creative campaign concepts based on this brief: [paste]."

18 — Documents

Rich-text documents for briefs, scripts, and notes — with external sharing and template support.

p.14
Creating a Document
  1. 1Go to Documents from the sidebar and click "New Document".
  2. 2Name the document and optionally link it to a client.
  3. 3Write using the rich-text editor — supports headings, bold, italic, bullet lists, numbered lists, tables, and code blocks.
  4. 4Documents auto-save as you type. There is no Save button — every change is saved immediately.
Sharing a Document
  1. 1Click the Share button in the document header.
  2. 2Toggle "Public Sharing" on — this generates a unique shareable link.
  3. 3Anyone with the link can read the document — no login required.
  4. 4Revoke access at any time by toggling sharing off. The link stops working immediately.
Document Types
TypeBest used for
BriefCampaign briefs, creative briefs, strategy briefs.
ScriptReel scripts, video production scripts, talking head outlines.
Strategy NoteStrategy summaries, quarterly direction documents.
SOPTeam processes, onboarding procedures, content guidelines.
Meeting NotesClient meetings, internal reviews, ideation sessions.
From Studio to Document: Studio tools (Content Studio, Hook Lab, Strategy) can save their AI outputs directly to a Document — no copy-pasting. Use this to preserve generated strategies or scripts in an editable, shareable format.
Practical Workflow
Account Manager creates a "Q3 Campaign Brief" document for a client, fills it in using a template, then shares the link with the client and the Creative Director. All stakeholders can read the same brief in real time — no PDF emailing, no version confusion, no "which file is the latest?" questions.
Document Best Practices
  • Link documents to the relevant client so they are easy to find later from the Client detail modal.
  • Use the document type field — it filters the document list so you can quickly find all briefs or all scripts.
  • For confidential strategy documents, keep sharing toggled off until you are ready to share.
  • Use headings (H1, H2, H3) to structure longer documents — this makes them readable on the shared link view.
  • Bold key terms and action items so skimmers can extract the essentials without reading everything.
  • The document URL in your browser can be bookmarked — it opens directly to that document.

AI Agent Tips
  • Fill in a detailed task description before running any AI agent — context is everything.
  • Run "Analyze Task" first to validate the brief before running "Write Copy".
  • Use thumbs up / down on AI outputs — this improves future outputs for that client over time.
  • AI responses are cached — same task + same context = instant result at zero extra cost.
  • Rate limit is 10 requests per user per minute — space out bulk AI runs across tasks.
  • Always select the client before using the AI Assistant — unscoped sessions give generic results.
Workflow Shortcuts
  • Esc key closes any open panel, modal, or slide-over from any page.
  • Click the NOVAX logo in the sidebar to return to Dashboard from anywhere in the app.
  • Filter chips on the Pipeline stay active — clear them when switching client context.
  • The floating My Tasks button (top header) is available on every page — check your queue without navigating away.
  • Documents auto-save — there is no Save button. Every keystroke is saved immediately.
  • Approval links can be resent at any time — just copy and share the same link from the Approval page.
Team Collaboration
  • Use task comments for all task-specific discussion — keeps context in one place and logged.
  • Check Workload before assigning new tasks — avoid overloading a teammate who is already at capacity.
  • Fill the Context Bank (Client → Intelligence) as you learn things — every win, objection, and signal makes future AI outputs smarter.
  • Send approval links 48+ hours before publish time to allow for revision cycles.
  • Use the Content Library to find and reuse high-performing posts as templates instead of starting from scratch.
  • Share Documents with external collaborators (freelancers, clients) using the public sharing link — no login required.

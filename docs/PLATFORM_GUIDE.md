# NOVAX Ops — Platform Guide
**For the Team · Launch Edition · June 2026**

---

## What Is NOVAX Ops?

NOVAX Ops is your agency's command centre. It replaces ClickUp, your scheduling tool, your client approval emails, your moderation inbox, and your reporting spreadsheets — all in one place.

Every piece of work — from a new brief to a published post — lives here. You can see what everyone is working on, move content through the pipeline, get AI help on any task, send clients a professional approval link, and pull a full performance report, all without leaving the platform.

---

## Getting In

### Login

Go to **perfumeexhibition.com** and enter your email and password. You will land on the dashboard immediately.

If it is your first time logging in, you will be taken to a short setup screen to enter your name, phone number, and set your password. This only happens once.

### If You Forget Your Password

Contact the admin. Password reset is handled through the admin panel — there is no self-serve reset link on the login screen.

---

## The Sidebar (Left Navigation)

The sidebar is your main way to move around the platform. It is always visible on the left. On mobile, tap the menu icon at the top to open it.

Each item in the sidebar takes you to a different page. Some items show a **badge** (a small number) — this means there is something waiting for your attention there (e.g. pending approvals, unread comments).

The sidebar shows only the pages you have access to. If you cannot see a page you expect, contact the admin.

## Pages

---

### Dashboard

**What it is:** Your daily overview. The first thing you see when you log in.

**What you will find:**

| Section | What it shows |
|---|---|
| KPI cards (top row) | Total clients · Active projects · Tasks due soon · Posts published this month |
| Activity chart | A graph of tasks completed across the current week |
| Social Performance | Live reach, impressions, engagement rate, likes, and comments pulled from all client accounts |
| Recent Posts | The latest published content per client, grouped by platform, with performance numbers |
| Pending items | A quick view of tasks that need attention |

**Key buttons:**

- **New Task** (top right of the header on every page) — creates a new task from anywhere.
- **Refresh** icon on the Social Performance section — pulls the latest numbers from the scheduling platform.

**Who uses it:** Everyone, every morning.

---

### Pipeline

**What it is:** The main task board. Every piece of work in the agency lives here as a card that moves through 10 stages from brief to reporting.

**The 10 stages (left to right):**

1. **Strategy** — brief received, not yet assigned
2. **Ideas** — concepts being explored
3. **Calendar** — approved direction, added to content calendar
4. **Copy** — writing in progress
5. **Design** — visual assets being created
6. **Review** — internal quality check
7. **Approval** — sent to client for sign-off
8. **Scheduled** — approved and queued to publish
9. **Published** — live on social media
10. **Reporting** — performance being tracked

**What you can do:**

- **Drag a card** left or right to move a task to a different stage.
- **Click a card** to open the Task Detail Panel (see below).
- **Filter** by client, assigned person, priority, or stage using the chips at the top.
- **Search** tasks using the search bar in the header.

**Task cards show:** Task title · Client name · Priority (colour-coded) · Assigned person · Due date.

**Key buttons:**

- **New Task** — opens the create task form. Fill in title, client, stage, assignee, due date, and description.

**Who uses it:** Everyone. This is the central workspace.

---

### Task Detail Panel

**What it is:** A slide-over that opens when you click any task card. This is where the actual work happens.

**Tabs:**

- **Details** — edit the task title, description, stage, priority, assignee, due date, and client.
- **AI Agents** — five AI tools built into every task (see below).
- **Copy Variants** — stores up to 3 written versions of the copy for this task.
- **Comments** — internal team conversation thread on this specific task.

**The 5 AI Agents:**

| Agent | What it does |
|---|---|
| Task Analyzer | Reads the brief and flags anything missing. Tells you the best approach for this task. |
| Copywriter | Writes 3 versions of the copy: aspirational, benefit-focused, and conversational. Arabic or English based on the client. |
| Researcher | Pulls market context, competitor gaps, and trending angles relevant to this task. |
| Asset Finder | Reads the brief and prepares a list of visual keywords to search in the asset library. |
| Post Caption | If you attach an image or video, this writes a caption specifically for that visual. |

Each AI output has a thumbs up / thumbs down button. Ratings feed back into future outputs — the AI learns what works for each client over time.

**Key buttons:**

- **Run** (on each agent) — triggers the AI. Results appear below. Cached after the first run, so re-running is instant and free.
- **Save Variant** — saves a written copy version to the Copy Variants tab.
- **Move Stage** — dropdown to change stage without dragging the card.

### Tasks (My Tasks)

**What it is:** A filtered view showing only the tasks assigned to you.

**What you can do:**

- See everything on your plate in one place.
- Filter by stage, priority, or client.
- Click any task to open the Task Detail Panel.

**Who uses it:** Everyone. Check this at the start of each day.

### Clients

**What it is:** The master list of all client brands.

**Client cards show:** Logo · Brand name · Active social platforms · Number of active projects.

**Clicking a client** opens a detailed panel with three tabs:

- **Overview** — client description, website, social links, brand identity summary (tone, target audience, key messages, primary platform).
- **Intelligence** — the client's AI memory. This is where wins, brand voice examples, audience insights, and objections are stored. The AI reads this before writing anything for this client. Your team can add entries manually via "Add Entry".
- **Tasks** — all active tasks for this client in a list.

**Crisis Mode** — a toggle on the client card (visible to senior roles). When turned on, it signals that this client is in a sensitive situation. A red banner appears everywhere this client is referenced to warn the team.

**Key buttons:**

- **New Client** — opens the 9-step onboarding wizard (see Workflows section).
- **Add Entry** (inside Intelligence tab) — saves a new piece of information about this client to the AI memory.
- **Design Brief** button — opens the visual identity form: canvas sizes, brand fonts, colours, motion notes, and video generation preferences.

**Who uses it:** Account managers, creative director, CEO, admin.

### Projects

**What it is:** Campaigns and project containers. Tasks belong to projects; projects belong to clients.

**Project cards show:** Project name · Client · Stage distribution bar (how many tasks are in each stage) · Overall progress percentage.

**What you can do:**

- Filter between active and completed projects.
- Click a project to see all tasks inside it.
- Create new projects and assign them to clients.

**Key buttons:**

- **New Project** — creates a project, assigns it to a client, sets timeline and goals.

**Who uses it:** Creative director, account managers, strategists.

### Publishing

**What it is:** Your content calendar and scheduling tool. All posts go through here to reach social media.

**Two views:**

- **Grid** — cards of all posts, sortable by status.
- **Calendar** — posts shown on the days they are scheduled. Use the arrow buttons to move between months.

**Post statuses:**

| Status | Meaning |
|---|---|
| Draft | Written, not yet scheduled |
| Scheduled | Queued and will publish at the set time |
| Published | Live on social media |
| Failed | Something went wrong — check the post and reschedule |

**What you can do:**

- **Compose** a new post manually.
- **Generate Calendar** — give the AI a brief and it drafts a full month of post ideas.
- **Edit** a scheduled post — change the caption, time, or platforms.
- **Delete** a draft or scheduled post.
- **Import from spreadsheet** — upload an Excel/CSV with post data to bulk-schedule.
- **Filter** by client, platform, or status.

**Compose form fields:**

- Client · Platforms (multi-select) · Caption · Language (English / Arabic / Both) · Media (upload or link from Google Drive) · Scheduled date and time

**Key buttons:**

- **Compose** — opens the compose form.
- **Generate Calendar** — opens the AI calendar generator.
- **Schedule** — sends the post to the scheduling platform at the set time.
- **Save as Draft** — saves without scheduling.

**Who uses it:** Social managers, copywriters, account managers.

---

### Approval

**What it is:** The client approval system. Send content to clients for sign-off through a professional, branded link — no email attachments.

**What you will find:**

- A list of all open approval requests, each showing the client name, number of posts, and current status.
- Status per post: Pending · Approved · Changes Requested.

**How approval works:**

1. You create an approval request and add the posts that need sign-off.
2. The system generates a unique link.
3. You send the link to the client (the system can email it automatically).
4. The client opens the link — no login needed — and reviews each post.
5. They can approve, request changes, or leave a note per post.
6. You see their decisions instantly in this panel.

**The client-facing portal** (`/approval/[token]`) is clean and branded. Clients see: the post image/video, caption, and platform — and three buttons: **Approve**, **Request Changes**, **Add Note**.

**Key buttons:**

- **New Approval Request** — select a client, add posts, set a deadline, send.
- **Copy Link** — copies the unique approval URL to share manually.
- **Send Email** — sends the link to the client email on file.
- **View Details** — see individual post decisions + client notes.

**Who uses it:** Account managers, social managers.

---

### Moderation

**What it is:** Your comment and DM inbox. Comments and messages from social media come in here via the connected messaging platform.

**What you will find:**

- A queue of unresolved comments and DMs, newest first.
- Each item shows: platform, account, the comment text, and how long it has been waiting.

**What you can do per item:**

- **AI Reply** — the AI drafts a suggested reply. You can edit it before sending.
- **Send** — sends the reply directly to the platform.
- **Escalate** — flags the item for a senior team member to handle.
- **Ignore** — archives the item without replying (spam, irrelevant).

**Filter options:** By platform (Instagram, Facebook, etc.) · By status (pending, replied, escalated).

> **Note at launch:** Receiving comments into the queue works. Sending replies via the platform API has a known limitation — Facebook public comments work; Instagram public comment replies via API are not supported (DMs work). This is a platform vendor constraint.

**Who uses it:** Social managers, account managers.

- **Filter** by client or file type.
- **Download** any asset.

Assets are stored securely and linked to clients where relevant.

**Key buttons:**

### Performance

**Three sections:**

1. **Post Analytics** — individual post performance (reach, likes, comments, saves, ER) synced from the scheduling platform.
2. **Competitor Tracking** — add competitor accounts and track their performance over time.
3. **AI Intelligence** — the AI identifies patterns from the data: which formats are winning, best times to post, what topics drive engagement.

**Key buttons:**

- **Sync** — pulls the latest stats from the scheduling platform.
- **Add Competitor** — enter a competitor handle to start tracking.

### CEO Hub

**What it is:** A private workspace for strategic decisions. Admin and CEO only.

**Three AI tools:**

1. **Strategy Analysis** — give the AI cross-client context and ask it to identify opportunities, risks, or patterns across the whole agency.
2. **Second Opinion** — describe a decision you are weighing and get a structured pros/cons + recommendation.
3. **Crisis Override** — when a client is in crisis mode, use this to get recommended messaging guardrails and a holding strategy.

**Who uses it:** CEO, admin.

### AI Assistant

**What it is:** A free-form AI chat that knows your clients and tasks.

**How to use it:**

- Type any question or request.
- Optionally, scope it to a specific client ("For Lumara — ") or task to get more relevant answers.
- Ask it to draft copy, brainstorm, summarise a brief, explain a trend, or help plan a campaign.

**What makes it different from a generic chatbot:** It has access to client brand identities, your team's task history, and the client context bank — so it gives answers tailored to your specific accounts, not generic advice.

**Who uses it:** Everyone.

### Documents

**What it is:** An in-platform document editor for briefs, SOPs, strategy docs, and anything your team writes collaboratively.

**What you can do:**

- **Create** a new document. Choose from blank or a template.
- **Edit** with a full rich text editor: headings, bullets, tables, bold/italic, links.
- **Share** — generate a public link so clients or external collaborators can view (read-only).
- **Export** — download as PDF.
- **Templates** — start from a pre-built structure for common document types.

**Who uses it:** Strategists, account managers, creative director.

---

### Creative Eval

**What it is:** Upload a piece of creative — image or video — and the AI scores it before it goes live.

**The scoring dimensions:**

| Dimension | What is being assessed |
|---|---|
| Brand Fit | Does this look and feel like the client's brand? |
| Hook Strength | Will it stop a scroll in the first 2 seconds? |
| Visual Quality | Composition, clarity, and production value |
| CTA Clarity | Is the next action obvious to the viewer? |
| Engagement Prediction | Likelihood of likes, comments, and shares based on patterns |

**Each dimension** gets a score and a short written explanation of why.

**When to use it:** Before submitting a design for client approval. Use the score to have a productive brief conversation with the designer rather than vague feedback.

**Who uses it:** Creative director, account managers, social managers.

## Common Workflows

---

### Workflow 1 — From Brief to Published Post

1. **Receive a brief** from the client or account manager.
2. Go to **Pipeline** → **New Task**. Fill in: client, title, description (paste the brief), stage = Strategy, assign to the strategist.
3. Strategist opens the task → runs **Task Analyzer** to check the brief is complete.
4. Task moves through Copy → Design → Review by dragging the card.
5. Copywriter opens the task → runs **Copywriter agent** → picks the best variant → saves it.
6. Task moves to **Approval** stage → account manager goes to the **Approval** page → creates an approval request → sends the link to the client.
7. Client approves → task moves to **Scheduled**.
8. Social manager goes to **Publishing** → **Compose** → pastes the approved copy, adds media, sets date/time → schedules it.
9. Post publishes automatically. Task moves to **Published**, then **Reporting**.

---

### Workflow 2 — Sending Content for Client Approval

1. Go to **Approval** page.
2. Click **New Approval Request**.
3. Select the client and choose which posts to include.
4. Set a review deadline.
5. Click **Send Email** — the client receives a branded email with a review link.
6. The client clicks the link, reviews each post, and marks Approve or Request Changes.
7. You see the decisions live in the Approval panel. If changes are requested, the notes appear inline.
8. Update the post, re-send if needed.

---

### Workflow 3 — Handling a Comment from a Follower

1. Open **Moderation** — the comment appears in the queue.
2. Read the comment. If it needs a reply, click **AI Reply**.
3. The AI drafts a response using the client's brand voice. Edit if needed.
4. Click **Send**.
5. If the comment is sensitive (complaint, crisis), click **Escalate** — it gets flagged for a senior team member.

---

### Workflow 4 — Monthly Client Report

1. Go to **Reports**.
2. Select the client and set the date range (e.g. June 1–30).
3. The platform pulls all performance data automatically.
4. Click **AI Narrative** — the AI writes a client-ready summary of the month's performance.
5. Review and edit the narrative.
6. Click **Export PDF** for a branded PDF, or **Export PPTX** for a presentation.
7. Send to the client.

---

### Workflow 5 — Adding a New Client

1. Go to **Clients** → **New Client**.
2. Work through the 9 steps:
   - **Identity** — name, industry, website, colour, logo
   - **Social** — which platforms, account handles
   - **Competitors** — main competing brands to track
   - **Brand** — description of the brand's personality
   - **Tone** — how the brand communicates (formal/casual, etc.)
   - **Audience** — who they are speaking to
   - **Strategy** — current goals and focus areas
   - **Goals** — measurable targets (followers, reach, engagement)
   - **Resources** — posting frequency, team assigned
3. Click Finish — client is saved and ready to use across all pages.

---

### Workflow 6 — Checking Your Team's Workload

1. Go to **Workload**.
2. See each team member's load bar.
3. If someone is overloaded (red bar), click their card to see their task list.
4. Go to **Pipeline**, find their overloaded tasks, and reassign to someone with capacity.

### Issues that need attention before Sunday

**1. ANTHROPIC_API_KEY is not set — CRITICAL**

All AI features (Task agents, Copywriter, AI Assistant, Reports narrative, Creative Eval, etc.) currently fall back to Gemini. The platform works but outputs will be Gemini quality, not Claude quality. If you want Claude for launch, set `ANTHROPIC_API_KEY` in the Vercel environment variables before Sunday.

Action: Go to Vercel → Project Settings → Environment Variables → add `ANTHROPIC_API_KEY`.

---

**2. Moderation replies — partial**

The comment queue receives and displays comments correctly. The AI reply suggestion works. However, **sending replies back to Instagram public comments** is blocked by the Instagram API (this is a permanent vendor constraint, not a code bug). Facebook public comments and DMs work. Set the team's expectation: use the platform to draft replies, then post them manually on Instagram if needed.

---

**3. Real-time updates — mostly live, a few pages still poll**

Pipeline, Approval, Moderation, and Notifications update live without a page refresh — if someone moves a task or a client submits an approval decision, you will see it instantly. The Dashboard, My Tasks list, and Publishing calendar still require a manual refresh to show the latest data. This does not break anything; it is a minor convenience gap for those specific pages.

---

**4. Studio tools — hide for launch**

Studio tools (Content Studio, Hook Lab, Strategy, Campaign Igniter, etc.) are fully built but need a separate training session before the team can use them productively. Gate them via page permissions for all non-admin users. Release in a dedicated Wave 2 session.
